# Rajasthali Tours Website

Express + EJS website configured for Render and Vercel deployment.

## Folder Structure

- `server.js` - Express application and SEO routes.
- `views/` - EJS templates and partials.
- `data/` - destinations, services, fleet, packages, and image manifest.
- `public/` - all files that must be served from the domain root.
- `public/rajasthalitours_images/` - source images used by the live site.
- `public/optimized/` - responsive WebP variants generated by `npm run build`.
- `scripts/` - image optimization and deployment checks.

## Domain Setup

The current live URL is:

`https://rajasthali-website.onrender.com`

This is the default `SITE_URL`, so canonical URLs, `robots.txt`, sitemap URLs, Open Graph URLs, and Search Console sitemap submissions match the Render domain.

When the final custom domain is live, set this environment variable in Render/Vercel and redeploy:

`SITE_URL=https://rajasthalitours.com`

## Vercel

Vercel serves static files from `public/**`, so image paths must stay root-relative, for example `/rajasthalitours_images/logo.png`.

The Search Console verification file is deployed from:

`public/google4eed84e29fda3a1e.html`
